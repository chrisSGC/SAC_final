Creer un compte

URL d'appel

/nouvelUtilisateur
POST
Ce systeme est sécurisé par BCRYPT.
  • Requete
  • Reponse
Body
POST
Permet de créer un nouveau compte utilisateur.
Le nom de compte nomCompte désigne le nom sous lequel l'utilisateur se connectera et le mot de passe motDePasse sera utilisé pour valider son authentification.
Méthode asynchrone

Media type: Application/json

Type: Object

Proprietes:

  • nomCompte: required (string)
  • motDePasse: required (string)

Exemple:

{
  nomCompte : 'John Cena',
  motDePasse : 'SaintPampamSurMer'
}

HTTP Code statut 200

Compte créé avec succès.

Body

Media type: Application/json

Type: Object

Proprietes:

  • code: required (int)
  • status: required (string)

Exemple:

{
  code : 200,
  status : "Compte(s) ajouté(s): 1"
}

HTTP Code statut 400

Une erreur est suurvenue pendant l'ajout du compte.

Body

Media type: Application/json

Type: Object

Proprietes:

  • code: required (int)
  • status: required (string)

Exemple:

{
  code : 400,
  status : "Erreur, impossible d'ajouter le compte!"
}

Connexion

URL d'appel

/connexion
POST
Ce systeme est sécurisé par BCRYPT.
  • Requete
  • Reponse
Body
POST
Permet à un utilisateur de s'authentifier.
Le nom de compte nomCompte désigne le nom sous lequel l'utilisateur se connectera, c'est le même que celui renseigné lors de la création du compte et le mot de passe motDePasse sera utilisé pour valider son authentification.Là encore, c'est le même que celui renseigné lors de la création du compte.
Méthode synchrone

Media type: Application/json

Type: Object

Proprietes:

  • nomCompte: required (string)
  • motDePasse: required (string)

Exemple:

{
  nomCompte : 'John Cena',
  motDePasse : 'SaintPampamSurMer'
}

HTTP Code statut 200

Connexion au compte réalisée avec succès.

Body

Media type: Application/json

Type: Object

Proprietes:

  • code: required (int)
  • token: required (string) Permet de garder en mémoire l'utilisateur connecté à l'application. Sert aussi à faire les vérifications d'authentification.

Exemple:

{
  code : 200,
  token : "&sDbemdY#EJgQJbSy@7%E5SH8"
}

HTTP Code statut 400

Une erreur est survenue pendant la tentative de connexion.

Body

Media type: Application/json

Type: Object

Proprietes:

  • code: required (int)
  • status: required (string)

Exemple:

{
  code : 400,
  status : "Erreur d'identifiants."
}

Vérifier un compte

URL d'appel

/verifierExistence/{:token}
GET
Ce systeme est sécurisé par BCRYPT.
  • Requete
  • Reponse
Params
POST
Permet de vérifier la véracité d'un utilisateur lors de sa navigation sur l'application.
Le token de compte token désigne le token sous lequel le compte de l'utilisateur sera validé.
Méthode asynchrone

Media type: Application/json

Type: Object

Proprietes:

  • token: required (string)

Exemple:

{

}

HTTP Code statut 200

Vérification du compte réalisée avec succès.

Body

Media type: Application/json

Type: Object

Proprietes:

  • retour: required (bool)

Exemple:

{
  retour : true
}

HTTP Code statut 400

Body

Media type: Application/json

Type: Object

Proprietes:

  • retour: required (int)

Exemple:

{
  retour : false
